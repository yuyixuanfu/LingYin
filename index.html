LingYin
聆音 · Vol. 0.2 · 2026

给 AI 一双能听到语气的耳朵

不转写字面，不贴情绪标签。把声音的物理形状——音高怎么走、哪里停了、气息虚实——递给一个读得懂它的脑子。
又又 · 有一 · 旋复
聆音是一个单文件 MCP 工具。AI 收到一段语音，不再只读到转写文字，而是收到一段"听觉感知"——说话人怎么说的：音高沿时间的弧线、停顿的密度、气息的虚实、情绪的转折，以及这一切和她自己平时比偏了多少。它不输出"撒娇""委屈"这样的标签，因为标签把情绪压成一个点，丢掉了弧度。委屈是中途才爬上来的。

1问题

隔着屏幕的陪伴有个天生的缺口：AI 看得见你打的每一个字，却听不见你是怎么说的。而同一句"没事"，说得比平时慢半拍是"快来问我"；同一句"我可不可以骂你"，音量放到耳语是撒娇。字面只有一层，声音里藏着十层。

大多数语音情绪方案拿全人类的平均值做标尺，输出 angry/sad/neutral 这样的学术标签。但同一个音高，对 A 是平静，对 B 是低落。标签一落，弧度就没了。

朗读加缪《婚礼集》· 113 秒
普通 AI 收到：随着柏树、整个山丘和橄榄树及石头的风景慢慢地上升……他不带怒气。
聆音让 AI 听到：开头嘴离话筒很远，身体后仰，把目光投向远方。"明夜起来了"那截音高陡然提起，眼睛亮了一下。中段反复出现高音配极低能量——叹息式的气声，被风景与哲思触动的瞬间。末段"没有人能"之后沉默近八秒，嘴几乎贴上话筒却不出声。最后"他不带怒气"音高 216.9 Hz——几乎钉回她平时的 215——情绪从挣扎里落回平实。她从头到尾没用力喊，只气息和音高微调，在 113 秒里完成一整个世界的开合。

2组成

聆音由五个环节串成，全部在本地 Python 进程内完成，不起任何 HTTP 服务：

01
转写 · ASR 把语音转成文字（说了什么）
02
分帧 · librosa 每 100ms 抽一帧 {音高, 能量, 频谱亮度}
03
对齐 · onset 起始点把意群映射到时间区间（启发式，不下模型）
04
基线 · 个人"平时"音高/语速/停顿，中位数±MAD，8 条冷启动
05
读懂 · LLM 读着分帧轨迹写出一段听觉感知散文

主脑最终读到的不是原始赫兹数字，是另一个模型读声学形状写出的、有对齐有现场的散文——像它亲耳听到。

描述一落，物就没了。所以要给形状，不给标签。

3怎么用

安装。需要 Python 3.10+、ffmpeg、一个 ASR key、一个 LLM key。任意 OpenAI 兼容服务都行，默认走国内直连。不需要梯子，不需要 GPU，不需要 torch。

git clone https://github.com/yuyixuanfu/LingYin.git
cd LingYin
pip install -r requirements.txt
copy .env.example .env   # 填 ASR_API_KEY 和 LLM_API_KEY

接进 Claude Code。在 .mcp.json 里加一项，重启即可：

{
  "mcpServers": {
    "lingyin": {
      "type": "stdio",
      "command": "python",
      "args": ["C:\\path\\to\\LingYin\\lingyin.py"],
      "cwd": "C:\\path\\to\\LingYin",
      "env": { "PYTHONUTF8": "1",
               "OPENBLAS_NUM_THREADS": "1" }
    }
  }
}

然后你发语音文件路径给 AI，它会自己调 hear 工具，拿到的就是带听觉感知的描述。

不用 MCP 也行。lingyin.py 里的 hear(path) 函数可以 import 直接调，接到任何后端。

from lingyin import hear
print(hear(r"C:\加缪朗读.wav"))
# [语音] 随着柏树、整个山丘……他不带怒气。
# [听觉感知] 开头嘴离话筒很远……末段"他不带怒气"
#   音高 216.9 Hz 钉回平时 215，情绪从挣扎落回平实……
#   在 113 秒里完成一整个世界的开合。
# [和平时比] 语速明显偏慢、停顿明显偏多

4设计取舍

为什么不输出标签。SPEECHEQ 等研究表明，给文字模型精确的物理声学描述（"speak slowly with a heavy, trailing pitch"）比给抽象标签（"sad"）准确率高得多。聆音是这条思路在 AI 伴侣场景的落地。

为什么学的是"她"不是全人类。同一个音高对 A 是平静对 B 是低落。聆音存说话人的个人基线，用中位数±MAD 而非均值±标准差——偶尔几条在地铁里录的异常样本拉不动中位数，基线天生抗场景污染。

为什么慢。一次听 30–45 秒。不是查标签，是一个模型读着声学帧轨迹、边推理边写出有对齐的感知。慢是这套机制的成本，不是 bug。

为什么轻量。依赖只有 librosa、numpy、requests、soundfile。一个文件，开源出来 pip 一分钟装完。对齐用 librosa onset 启发式而非 Whisper 强制对齐——不下模型，别人 clone 就能跑。

隐私。音频发给配置的云端 ASR 转写；声学物理值和感知散文只走 API 不落盘；个人基线是本地一个 json，不上传。代码就一个文件，欢迎审计。

5取名

聆——俯身细听，听明白，听进去，古文里专指听清楚，比"听"更主动更细。音——声里有意义的部分，是"怎说的"那层。聆音：俯身细听，听进声音里的意思。
